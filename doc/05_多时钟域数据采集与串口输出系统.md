## 二、系统架构

```
                    ┌──────────────────────────────────────────────────┐
                    │            sampler_uart_top                      │
                    │                                                  │
 clk_fast(200MHz)──►│  ┌──────────────┐    ┌────────────┐             │
                    │  │ data_sampler  │───►│ async_fifo  │            │
 rst_n ────────────►│  │  (200MHz域)   │    │ (跨时钟域)  │            │
                    │  └──────────────┘    └─────┬──────┘            │
                    │                            │                    │
  clk_sys(50MHz)───►│                      ┌─────▼──────┐            │
                    │                      │  uart_tx    │───► tx_out │
                    │                      │  (50MHz域)  │            │
                    │                      └────────────┘            │
                    └──────────────────────────────────────────────────┘
```

数据流：`data_sampler` 在 200 MHz 时钟域产生采样数据，写入异步 FIFO；`uart_tx` 在 50 MHz 时钟域从 FIFO 读取数据，按 UART 协议串行发送。

## 三、模块设计

### 3.1 sampler_uart_top — 顶层模块

将三个子模块连接，并对外暴露时钟、复位与 UART 输出。

**参数：**

| 参数名       | 默认值      | 说明                     |
| ------------ | ----------- | ------------------------ |
| DATA_WIDTH   | 8           | 数据位宽                 |
| BAUD_RATE    | 115200      | UART 波特率              |
| CLK_SYS_FREQ | 50_000_000 | 系统时钟频率（Hz）       |

**端口：**

| 信号名   | 方向   | 位宽 | 说明                           |
| -------- | ------ | ---- | ------------------------------ |
| clk_fast | input  | 1    | 快时钟，200 MHz                |
| clk_sys  | input  | 1    | 系统时钟，50 MHz               |
| rst_n    | input  | 1    | 全局异步复位，低有效           |
| tx_out   | output | 1    | UART 串行数据输出              |
| fifo_full| output | 1    | FIFO 满指示（调试用）          |
| fifo_empty| output| 1    | FIFO 空指示（调试用）          |

### 3.2 data_sampler — 数据采集模块

工作在 200 MHz 时钟域。内部维护一个递增计数器作为模拟采样数据源。当 FIFO 未满时，以可控速率将数据写入 FIFO。

**参数：**

| 参数名      | 默认值 | 说明                               |
| ----------- | ------ | ---------------------------------- |
| DATA_WIDTH  | 8      | 输出数据位宽                       |
| SAMPLE_DIV  | 20000  | 采样分频系数（每 N 个时钟产生一笔） |

**端口：**

| 信号名     | 方向   | 位宽       | 说明                         |
| ---------- | ------ | ---------- | ---------------------------- |
| clk        | input  | 1          | 200 MHz 时钟                 |
| rst_n      | input  | 1          | 异步复位，低有效             |
| data_out   | output | DATA_WIDTH | 采样数据输出                 |
| data_valid | output | 1          | 数据有效指示                 |
| fifo_full  | input  | 1          | FIFO 满信号（反压）          |

**功能描述：**
- 内部分频计数器每 `SAMPLE_DIV` 个时钟周期产生一次数据
- 数据为 8-bit 递增计数器值（0x00 ~ 0xFF 循环），串口输出形如 `00 01 02 ... FE FF 00 01 ...`，接收端通过观察数值是否连续递增即可判断数据完整性
- 当 `fifo_full` 为高时暂停产生数据（反压机制）
- `data_valid` 仅在产生有效数据且 FIFO 未满时拉高一个时钟周期

**`SAMPLE_DIV` 取值说明：**

UART 115200 baud、8N1 格式下，每字节需 10 bit，理论吞吐量为 115200 / 10 = 11520 字节/秒，即每字节约 86.8 μs。在 200 MHz 时钟下对应约 17361 个时钟周期。`SAMPLE_DIV` 取 20000（对应每笔间隔 100 μs，即 10000 字节/秒）略低于 UART 吞吐上限，确保 FIFO 不会被填满，串口输出连续无断裂。

### 3.3 异步 FIFO — Xilinx FIFO Generator IP

跨时钟域 FIFO，写端工作在 `clk_fast`（200 MHz），读端工作在 `clk_sys`（50 MHz）。不手写 RTL，使用 SynthPilot MCP 工具配置一个 Xilinx FIFO Generator IP，配置要求如下：

| 配置项       | 值           | 说明                           |
| ------------ | ------------ | ------------------------------ |
| IP 实例名    | async_fifo   |                                |
| 数据位宽     | 8            | 与 data_sampler 输出一致       |
| 深度         | 16           | 教程演示用，无需太大           |
| 异步模式     | 是           | 写时钟 200 MHz，读时钟 50 MHz  |
| 读模式       | First Word Fall Through | 便于与下游握手逻辑对接 |
| 存储类型     | Block RAM    | 默认即可                       |

IP 生成后，其典型端口为 `wr_clk`、`rd_clk`、`din`、`wr_en`、`full`、`dout`、`rd_en`、`empty` 等，顶层例化时按实际生成的端口连接。

### 3.4 uart_tx — UART 发送模块

工作在 50 MHz 时钟域。从 FIFO 读取数据，按标准 UART 协议（8N1）串行发送。

**参数：**

| 参数名    | 默认值      | 说明               |
| --------- | ----------- | ------------------ |
| CLK_FREQ  | 50_000_000  | 系统时钟频率（Hz） |
| BAUD_RATE | 115200      | 波特率             |

**端口：**

| 信号名     | 方向   | 位宽 | 说明                           |
| ---------- | ------ | ---- | ------------------------------ |
| clk        | input  | 1    | 50 MHz 时钟                    |
| rst_n      | input  | 1    | 异步复位，低有效               |
| tx_data    | input  | 8    | 待发送数据                     |
| tx_valid   | input  | 1    | 数据有效（握手信号）           |
| tx_ready   | output | 1    | 发送器就绪（握手信号）         |
| tx_out     | output | 1    | UART 串行输出（空闲态为高）    |

**功能描述：**
- 空闲时 `tx_out` 保持高电平，`tx_ready` 为高
- 当 `tx_valid` 与 `tx_ready` 同时为高时，锁存数据并开始发送
- 发送帧格式：1 起始位（低）+ 8 数据位（LSB first）+ 1 停止位（高）
- 内部波特率分频计数器从系统时钟分频产生 bit 时钟
- 发送过程中 `tx_ready` 拉低，发送完成后恢复

## 四、顶层连接逻辑

顶层模块 `sampler_uart_top` 的内部连接关系：

```
data_sampler.data_out      ──► async_fifo.din
data_sampler.data_valid    ──► async_fifo.wr_en（受 wr_rst_busy 门控）
async_fifo.full | wr_rst_busy ──► data_sampler.fifo_full

async_fifo.dout            ──► uart_tx.tx_data
~async_fifo.empty & ~rd_rst_busy ──► uart_tx.tx_valid
uart_tx.tx_ready & ~empty & ~rd_rst_busy ──► async_fifo.rd_en
```

读端握手逻辑：FIFO 非空时将数据送入 UART 发送器，需正确处理 FIFO 读使能与 UART 握手的配合，避免对空 FIFO 发起读操作。注意 `wr_rst_busy` / `rd_rst_busy` 期间禁止读写。

## 六、仿真验证 Checklist

Testbench 例化完整的 `sampler_uart_top`（内含 `data_sampler`、`async_fifo`、`uart_tx`），并在外部编写 UART 接收模型（`uart_rx_model`）对串行输出 `tx_out` 进行采样解码。对以下场景逐项验证并自动判定 PASS/FAIL：

### 6.1 测试用例

| 编号   | 测试场景                           | 通过条件                                                                 |
| ------ | ---------------------------------- | ------------------------------------------------------------------------ |
| TC-01  | 复位初始状态                       | 复位释放后，`tx_out` 为高（空闲态），`fifo_empty` 为高，`fifo_full` 为低 |
| TC-02  | 单字节 UART 发送帧格式            | UART 接收模型解码出完整帧：1 起始位（低）+ 8 数据位（LSB first）+ 1 停止位（高） |
| TC-03  | 数据完整性（连续递增验证）         | UART 接收模型连续接收至少 16 字节，数值为 0x00→0x01→…→0x0F 连续递增     |
| TC-04  | 跨时钟域数据一致性                 | 200 MHz 域写入 FIFO 的数据与 50 MHz 域 UART 发出的数据逐字节比对，无丢失无错位 |
| TC-05  | FIFO 反压机制                      | 将 `SAMPLE_DIV` 设为极小值（如 2）使写入速率远超读取速率，验证 `fifo_full` 拉高后 `data_sampler` 暂停写入，FIFO 不溢出 |
| TC-06  | FIFO 空状态处理                    | 将 `SAMPLE_DIV` 设为极大值使 FIFO 长期为空，验证 `uart_tx` 不发送虚假数据，`tx_out` 保持空闲高电平 |
| TC-07  | 波特率精度                         | 测量 UART 输出每个 bit 的持续时间，与理论值（1/115200 s ≈ 8.68 μs）偏差不超过 ±2% |
| TC-08  | 长时间连续运行                     | 系统连续运行并成功接收至少 256 字节（覆盖计数器 0x00~0xFF 完整一轮），全部数据正确 |
| TC-09  | 复位恢复                           | 系统正常运行中途触发复位，释放后系统重新正常工作，UART 输出有效数据       |
| TC-10  | FIFO 满到非满恢复                  | 触发 FIFO 满后恢复正常速率，验证数据流恢复畅通，后续字节仍连续递增       |

### 6.2 UART 接收模型（uart_rx_model）

Testbench 内需实现一个行为级 UART 接收模型，用于自动解码 `tx_out` 信号：

- 检测起始位下降沿后，以波特率中点采样 8 个数据位
- 校验停止位为高电平
- 将每字节解码结果存入队列或数组，供各测试用例比对

### 6.3 Testbench 参数化

为便于不同测试场景复用同一 testbench，建议通过参数覆盖调整行为：

| 参数名               | 默认值   | 用途                                       |
| -------------------- | -------- | ------------------------------------------ |
| TB_SAMPLE_DIV        | 200      | 覆盖 `data_sampler` 的 `SAMPLE_DIV`（加速仿真） |
| TB_BAUD_RATE         | 1000000  | 覆盖波特率（1 Mbaud 加速仿真）             |
| TB_NUM_BYTES         | 256      | TC-08 长时间运行接收的目标字节数           |
| SIM_TIMEOUT_US       | 50000    | 仿真超时保护（μs），超时后强制结束并报 FAIL |

### 6.4 Testbench 输出要求

- 每个测试用例打印 `[PASS] TC-xx: 描述` 或 `[FAIL] TC-xx: 描述 — 期望值=xx, 实际值=xx`
- 全部完成后打印汇总：`=== x/10 PASSED ===`
- 使用 `$dumpfile` / `$dumpvars` 生成波形文件，以便 Vivado 查看
- UART 接收模型每成功解码一个字节打印 `[UART_RX] @%0t: 0x%02X`，便于调试

### 6.5 仿真加速建议

由于 UART 115200 波特率下每字节耗时约 86.8 μs，256 字节需 ~22 ms 仿真时间，在 200 MHz 时钟下约 440 万周期。为加速仿真：
- 可在仿真时将 `BAUD_RATE` 提高（如 1_000_000）并同步调整 `SAMPLE_DIV`
- 或减少 `TB_NUM_BYTES` 做快速回归，仅在最终验证时使用完整参数

## 八、综合后分析

综合完成后，打开综合后设计（`open_synthesized_design`），使用 SynthPilot MCP 工具进行分析。

> **关于分析阶段的说明：** 综合后可获得准确的资源利用率和网表级结构信息，时序分析基于估算线延迟（非实际布线），结果可用于早期问题发现但不作为签收依据。拥塞分析、布线状态、精确功耗估算需布局布线后才可获得，本阶段不涉及。

### 8.1 资源利用率分析 — 综合后可用，数据准确

- `report_utilization`：查看 LUT、FF、BRAM、DSP、IO 等资源使用量
- `report_utilization_hierarchical`：按模块层次分解，定位各子模块资源开销
- 确认总资源占用在 XC7Z010 容量范围内

### 8.2 时序分析 — 综合后可用，基于估算线延迟

- `report_timing_summary`：WNS / TNS / WHS / THS 汇总，确认各时钟域是否满足约束
- `analyze_critical_paths`：查看最差路径的逻辑级数与延迟分解
- `analyze_clock_domain_timing`：时钟对之间的 WNS 矩阵，关注跨时钟域路径
- `report_high_fanout_nets`：高扇出网络，识别潜在布线瓶颈
- **注意：** 综合后时序使用估算线延迟，实际 slack 在布局布线后会变化；此阶段主要用于发现大幅违例

### 8.3 时钟网络分析 — 综合后可用，数据准确

- `report_clock_networks`：确认 Clocking Wizard 输出的时钟树结构
- `report_clock_utilization`：检查 BUFG 等时钟资源分配
- `report_clock_interaction`：时钟域交互矩阵，确认 CDC 路径标识正确

### 8.4 CDC 分析 — 综合后可用

- `report_cdc`：跨时钟域路径报告
- `report_cdc_details`：详细的同步器识别与 CDC 路径分类
- 确认异步 FIFO 内部的 CDC 路径已被正确处理，无遗漏的裸跨时钟域信号

### 8.5 设计规则与方法学检查 — 综合后可用（部分规则）

- `report_drc`：设计规则检查，关注时钟、复位、I/O 相关告警
- `report_methodology`：方法学检查，发现约束完整性与最佳实践问题
- `check_timing`：检查约束覆盖是否完整（无未约束端口、无缺失时钟）
- **注意：** 部分 DRC 规则（如布线相关）仅在布局布线后生效，综合后检查覆盖的是子集
