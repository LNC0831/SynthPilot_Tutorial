# 设计文档：多时钟域数据采集与串口输出系统

## 1. 模块功能概述

本系统实现了一套多时钟域数据采集与 UART 串口输出方案，包含三个核心模块：

- **data_sampler**：工作在 200 MHz 快时钟域，以可配置速率产生 8-bit 递增计数数据
- **async_fifo**：Xilinx FIFO Generator IP，实现 200 MHz → 50 MHz 的异步跨时钟域数据传递
- **uart_tx**：工作在 50 MHz 系统时钟域，按 8N1 格式将数据串行发送

## 2. 顶层架构框图

```
                    ┌──────────────────────────────────────────────────┐
                    │            sampler_uart_top                      │
                    │                                                  │
 clk_fast(200MHz)──►│  ┌──────────────┐    ┌────────────┐             │
                    │  │ data_sampler  │───►│ async_fifo  │            │
 rst_n ────────────►│  │  (200MHz域)   │    │ (跨时钟域)  │            │
                    │  └──────────────┘    └─────┬──────┘            │
                    │                            │                    │
  clk_sys(50MHz)───►│                      ┌─────▼──────┐            │
                    │                      │  uart_tx    │───► tx_out │
                    │                      │  (50MHz域)  │            │
                    │                      └────────────┘            │
                    └──────────────────────────────────────────────────┘
```

内部连接关系：
- `data_sampler.data_out` → `async_fifo.din`
- `data_sampler.data_valid` → `async_fifo.wr_en`（受 `wr_rst_busy` 门控）
- `async_fifo.full | wr_rst_busy` → `data_sampler.fifo_full`
- `async_fifo.dout` → `uart_tx.tx_data`
- `~async_fifo.empty & ~rd_rst_busy` → `uart_tx.tx_valid`
- `uart_tx.tx_ready & ~empty & ~rd_rst_busy` → `async_fifo.rd_en`

## 4. 异步 FIFO IP 配置说明

使用 Xilinx FIFO Generator IP (`fifo_generator v13.2`)，实例名 `async_fifo`。

| 配置项   | 值                          |
| -------- | --------------------------- |
| 数据位宽 | 8 bit                       |
| 深度     | 16                          |
| 时钟模式 | Independent Clocks (异步)   |
| 读模式   | First Word Fall Through     |
| 存储类型 | Block RAM                   |
| 复位     | 高有效 (rst)                |

关键端口：`rst`（高有效，接 `~rst_n`）、`wr_clk`、`rd_clk`、`din[7:0]`、`wr_en`、`rd_en`、`dout[7:0]`、`full`、`empty`、`wr_rst_busy`、`rd_rst_busy`。

FWFT 模式下，当 `empty` 去断言时 `dout` 即有效，无需额外读延迟。`wr_rst_busy` / `rd_rst_busy` 期间不可进行读写操作。

## 5. UART 发送状态机描述

```
    ┌──────┐  tx_valid & tx_ready   ┌───────┐
    │ IDLE │──────────────────────►│ START │
    │tx=1  │                       │tx=0   │
    │ready=1                       │ready=0│
    └──┬───┘◄──────────────────────└───┬───┘
       │         baud_tick              │ baud_tick
       │                               ▼
       │                          ┌────────┐
       │    baud_tick & bit=7     │  DATA  │
       │◄─────────────────────────│tx=d[n] │
       │         (via STOP)       │  n=0~7 │
       │                          └───┬────┘
       │                              │ baud_tick & bit<7
       │         ┌──────┐            │ (loop)
       └─────────│ STOP │◄───────────┘
        baud_tick│tx=1  │
                 └──────┘
```

- **IDLE**：`tx_out=1`, `tx_ready=1`。检测到 `tx_valid & tx_ready` 时锁存数据，输出起始位
- **START**：输出低电平（起始位），持续 1 个 bit 时间
- **DATA**：LSB first 逐位输出 8 个数据位，每位持续 1 个 bit 时间
- **STOP**：输出高电平（停止位），持续 1 个 bit 时间，完成后返回 IDLE

波特率分频：`BAUD_DIV = CLK_FREQ / BAUD_RATE`，每 BAUD_DIV 个时钟周期产生一个 bit 时钟。
