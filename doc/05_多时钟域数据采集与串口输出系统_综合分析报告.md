## 5. DRC / 方法学检查结果及处理说明

### 5.1 DRC 检查

| 规则 ID    | 严重级别 | 数量 | 描述                              |
| ---------- | -------- | ---: | --------------------------------- |
| REQP-1840  | Warning  |    6 | RAMB18 async control check        |
| ZPS7-1     | Warning  |    1 | PS7 block required                |

**处理说明：**

- **REQP-1840**（6 条）：BRAM 的控制引脚（ENARDEN/ENBWREN/WEBWE）由具有异步复位的寄存器驱动（`data_valid_reg` / `tx_ready_reg`）。这是异步 FIFO + 异步复位设计的固有特征。FIFO IP 内部已有复位同步机制（`wr_rst_busy` / `rd_rst_busy`），实际使用中 BRAM 不会在复位释放瞬间被错误写入。此告警可安全忽略。
- **ZPS7-1**（1 条）：Zynq 器件要求例化 PS7 模块以正确配置默认行为。本设计仅使用 PL 部分，不涉及 PS 侧功能。若后续需要下载到实际 Zynq 开发板，需添加 PS7 例化或使用 Block Design 集成。此告警在纯 PL 教学场景下可接受。

### 5.2 方法学检查

| 规则 ID    | 严重级别 | 数量 | 描述                                     |
| ---------- | -------- | ---: | ---------------------------------------- |
| LUTAR-1    | Warning  |    1 | LUT drives async reset alert             |
| TIMING-18  | Warning  |    2 | Missing input or output delay            |
| XDCC-1     | Warning  |    1 | Scoped Clock constraint overwritten      |
| XDCC-7     | Warning  |    1 | Scoped Clock overwritten on same source  |

**处理说明：**

- **LUTAR-1**：组合逻辑（`locked & sys_rst_n`）驱动异步复位引脚，LUT 毛刺可能导致意外复位。这是板级复位组合的常见模式，实际中 `locked` 信号在稳定后不会产生毛刺。如需更严格的设计，可在 `locked` 信号后加同步器。教学场景下可接受。
- **TIMING-18**（2 条）：`sys_rst_n` 缺少 input delay，`uart_tx` 缺少 output delay。复位信号为异步信号，无需精确时序约束；UART TX 为低速接口（115200 baud），时序裕量远超 IO 延迟。教学场景下可安全忽略。
- **XDCC-1 / XDCC-7**：用户 XDC 中的 `create_clock` 覆盖了 Clocking Wizard IP 内部的同名时钟约束。两者参数完全一致（均为 20 ns 周期），不影响时序分析结果。可删除用户 XDC 中的 `create_clock` 行以消除此告警，但保留也不会导致问题。

## 7. 已知问题与后续优化建议

### 7.1 已知问题

1. **ZPS7-1 告警**：未例化 PS7 模块。如需在 Zynq 开发板上实际运行，需通过 Block Design 添加 ZYNQ7 Processing System IP 或手动例化 PS7 原语。
2. **异步复位驱动 BRAM**：REQP-1840 告警。FIFO IP 的 `wr_rst_busy` / `rd_rst_busy` 机制已确保安全，但若追求零告警，可考虑将 data_sampler 和 uart_tx 改用同步复位。

### 7.2 后续实现阶段建议

1. **布局布线后重新验证时序**：当前时序基于估算线延迟，实际 slack 可能变化。200 MHz 域 WNS = 1.319 ns 裕量适中，需在实现后确认。
2. **消除时钟约束重复告警**：可删除 `board_top.xdc` 中的 `create_clock` 行，依赖 Clocking Wizard IP 内置约束。
3. **添加 IO delay 约束**：如需严格时序签收，可为 `sys_rst_n` 和 `uart_tx` 添加 `set_false_path` 或 `set_input_delay` / `set_output_delay`。
4. **功耗分析**：需在布局布线后使用 `report_power` 获取准确功耗估算。
